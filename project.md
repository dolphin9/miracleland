## 已知问题与技术债务

### 高优先级问题

#### 1. 代码重复（导航栏和页脚）
- 导航栏和页脚代码在 Index.vue, WorldView.vue, OCView.vue, CreationsView.vue 中重复定义，维护成本高，建议提取为 HeaderNav.vue 和 Footer.vue 组件。

#### 2. 硬编码样式值
- 颜色、间距、尺寸等设计值直接写在组件 <style> 块中，未提取为 CSS 变量，建议在 src/style.css 中定义全局 CSS 变量。

#### 3. 缺少 API 集成层
- OCView 和 CreationsView 使用硬编码的模拟数据，缺少 Axios 和 API 服务层，需安装 Axios 和 Pinia 并创建相关服务和 store。

#### 4. 缺少详情页路由
- OC 和 Creation 卡片有“查看详情”按钮，但目标路由不存在，需创建 OCDetail.vue、CreationDetail.vue 组件及动态路由。

### 中优先级问题

#### 5. TypeScript 未使用
- TypeScript 已安装但项目使用纯 JavaScript，可选迁移。

#### 6. 缺少错误处理和加载状态
- API 调用时没有错误处理、超时处理或加载指示器，建议在 API 服务层和组件中补充。

#### 7. Sass 已安装但未使用
- Sass 已安装但所有样式使用纯 CSS，可选统一使用 CSS 变量。

### 低优先级问题

#### 8. 缺少单元测试和集成测试
- 项目中未配置测试框架（Vitest 或 Jest）。

#### 9. 部署流程未验证
- 部署指南存在，但实际生产构建和部署未测试。

### 建议修复顺序

1. 立即处理：问题 1、2（提升代码质量）
2. Phase 3 前：问题 3、4（解锁 API 集成）
3. Phase 4：问题 6（优化用户体验）
4. 可选：问题 5、7（技术优化）
5. Phase 5：问题 8、9（测试和部署）
